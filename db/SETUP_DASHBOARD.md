# SEAM 診断ダッシュボード — オーナー様向け 設定手順

カウンセリング（髪質診断）の集計結果を見るための管理画面です。
**個人情報は一切保存しません**（氏名・連絡先・Cookieなし）。診断タイプ・悩み・ダメージ傾向・予約導線などの「件数」だけを匿名で集計します。

- 画面URL: **https://seam.site/admin.html**
- 検索エンジンには出ません（noindex）。URLとパスキーを知っている人だけが見られます。

---

## 仕組み（全体像）

```
診断ページ(finder) ──イベント送信──▶ /api/ev ──保存──▶ Cloudflare D1（seam-db）
                                                              │
管理画面 admin.html ──パスキー認証──▶ /api/admin/stats ──集計──┘
```

コード側（`db/schema.sql` / `functions/api/ev.js` / `functions/api/admin/stats.js` / `admin.html`）はデプロイ済みです。
あとは **Cloudflare 側で「保存先(D1)を作って繋ぐ」設定だけ** が必要です（5分ほど・下記）。

> 設定が終わるまでは、管理画面に「あと少しで集計が始まります」という案内が出ます（壊れているわけではありません）。

---

## 設定手順（Cloudflare ダッシュボード）

### ① データベース(D1)を作る
1. Cloudflare にログイン → 左メニュー **Workers & Pages** → **D1 SQL Database**
2. **Create** をクリック → 名前を **`seam-db`** にして作成

### ② テーブルを作る（スキーマ投入）
1. 作成した `seam-db` を開く → **Console** タブ
2. リポジトリの **`db/schema.sql`** の中身をすべてコピーして貼り付け → **Execute**
   （`events` テーブルとインデックスが作られます）

### ③ サイトにD1を繋ぐ（バインディング）
1. **Workers & Pages** → サイト **seam-website**（SEAM本体）→ **Settings** → **Functions**
2. **D1 database bindings** → **Add binding**
   - Variable name: **`DB`**（大文字・この名前のまま）
   - D1 database: **`seam-db`**
3. 保存

### ④ ダッシュボードのパスキーを決める
1. 同じ **Settings** → **Environment variables**（Production）
2. **Add variable**
   - Name: **`ADMIN_KEY`**
   - Value: **任意の長いパスキー**（例: 英数字20文字以上のランダムな文字列）
   - 可能なら **Encrypt（暗号化）** を選択
3. 保存

> このパスキーが管理画面のログインパスワードになります。**メンバーへ共有する際は安全な方法で。**

### ⑤ 再デプロイ
- **Deployments** → 最新のデプロイの **⋯** → **Retry deployment**（または次回のpushで自動反映）
- これで設定が反映され、診断のたびにデータが溜まり始めます

## 使い方

1. **https://seam.site/admin.html** を開く
2. ④で決めた **パスキー** を入力 → 「表示する」
3. 上部の期間（30日 / 90日 / 1年）で絞り込み
4. **CSV書出** で集計をダウンロード（Excel等で分析・メーカー共有に）

### 見られる集計
- **KPI**: 診断完了数 / 開始数 / 完了率 / 予約・CTA遷移率
- **27髪格タイプ 分布**（どのタイプが多いか）
- **悩み（主訴）分布**（頭皮 / うねり / ボリューム / ダメージ / 乾燥 / カラー / 熱）
- **ダメージ履歴 Tier**（軽い / 中 / しっかり）
- **性別 / 診断モード（かんたん・くわしい）**
- **予約・誘導クリック**（サロン予約・オンラインショップ等のどこへ進んだか）
- **日次トレンド**（診断完了の推移）

## よくある質問

**Q. 個人情報は入りますか？**
いいえ。氏名・連絡先・メールなどは一切送信・保存していません。Cookieも使いません。集計に必要な匿名の傾向データ（診断タイプ・悩み・件数など）だけです。

**Q. パスキーを変えたい / 漏れたかも**
④の `ADMIN_KEY` を新しい値に更新して再デプロイすれば、即座に古いパスキーは無効になります。

**Q. データを消したい / リセットしたい**
D1 Console で `DELETE FROM events;` を実行すると全件削除できます（戻せないのでご注意ください）。

## 補足（開発メモ・任意）

- **収集の堅牢性**: `/api/ev` は計測でサイトを壊さないよう、保存先が未設定でも常に正常応答します（D1未接続時はCFのリアルタイムログにのみ出力）。
- **拡張の余地（v2・任意）**: 現状は「診断タイプ・主訴・ダメージTier・性別・モード・予約導線」を集計します。さらに踏み込んで「選んだ悩みの生データ（複数選択）」「髪の長さ」「相性の出たブランド」まで集計したい場合は、finder側のイベントに `meta` を足す小改修で対応可能です（スキーマの `meta` 列が予約済み）。ご要望あれば対応します。
- **スパム対策（任意）**: `/api/ev` は収集のため公開エンドポイントです（PIIは無いので漏洩リスクはありません）。いたずら投稿で件数が膨らむのが気になる場合は、簡易レート制限やトークンを後付けできます。
