`damage >= 7` のユーザーに対して、キャラ別アフィニティに関係なく
**集中補修系商品が必ずスコアトップ層に並ぶ** 仕組みを追加。

- Tokio Premium SH / TR
- Aujua Aquavia SH / TR / Mask
- Elujuda Extrarepair Serum / Milky
- ViEqualite UH SH
- Aujua Diorum SH / TR

**実装箇所:**
- `/tmp/finder_simulator.py` (Python 検証)
- `finder.html` L2390 付近 `pickDeepProducts` 内
  - `DAMAGE_RESCUE_IDS` Set 追加
  - `damageRescueBoost(p)` 関数追加 ( +35 score )
  - スコア合計に `+ dmgrep` を組み込み

### 既存の Fix #3: BLEACH RESCUE BOOST と棲み分け

| Fix | トリガー | 加点 | 用途 |
|-----|---------|------|------|
| #3 BLEACH | `bleachHistory >= 4` | +40 | 強ブリーチ専用補修 (Wella Ultime Smooth 含む) |
| #9 DAMAGE | `damage >= 7` | +35 | ブリーチ以外でも高ダメージなら発火 |

両者は **独立に加算** されるため、強ブリーチ＋高ダメージのユーザーは
両方の補修系商品が同時に上位へ。
